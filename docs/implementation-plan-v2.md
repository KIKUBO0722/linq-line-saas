## フェーズ別実装企画

### Phase 1: 移行の壁を壊す（1セッション）
> 代理店が「既存クライアントを移せる」と確信できる状態にする

#### 1-1. CSVエクスポート（友だち + タグ）
- **仕様**: GET /api/v1/friends/export?format=csv
- **出力カラム**: 表示名, LINE UserID, ステータス, タグ（カンマ区切り）, スコア, 追加日, 言語, 流入元, カスタムフィールド（JSON）
- **実装量**: API 1エンドポイント（~50行）、フロント ボタン1個
- **工数**: 小

#### 1-2. CSVインポート（友だち + タグ自動作成）
- **仕様**: POST /api/v1/friends/import (multipart/form-data)
- **対応フォーマット**: LinQ標準CSV / エルメ互換 / Lステップ互換（ヘッダー自動判定）
- **処理**: 行ごとにupsert、存在しないタグは自動作成、エラー行はスキップしてレポート返却
- **バリデーション**: LINE UserIDの重複チェック、必須カラム検証
- **実装量**: API ~150行、フロント アップロードUI + プレビュー + 結果表示
- **工数**: 中

---

### Phase 2: AIで「構築を代替する」（1-2セッション）
> デモで「30万円分の構築が30秒」を証明する

#### 2-1. AIシナリオ自動生成（B3）★最重要デモ機能
- **仕様**: 業種（美容/不動産/飲食/EC/コンサル等）と目的（新規育成/リピート促進/セミナー誘導等）を選択
- **出力**: 5〜10通のステップ配信シナリオ（各メッセージのテキスト + 配信間隔 + 条件分岐の提案）
- **AI処理**: Claude APIにプロンプト → 構造化JSON → stepScenarios + stepMessages に一括保存
- **UI**: ウィザード形式（業種選択→目的選択→生成→プレビュー→「このまま保存」ボタン）
- **差別化ポイント**: Lステップの認定コンサルが手動で設計する部分をAIが即時生成。業種別のベストプラクティスをプロンプトに組み込む
- **実装量**: API ~200行（AI生成 + 一括保存）、フロント ウィザードUI ~400行
- **工数**: 中〜大

#### 2-2. AIチャットサジェスト（B4）
- **仕様**: 1:1チャット画面で、受信メッセージに対するAI返信案を3パターン表示
- **UI**: チャット入力欄の上に「AI提案」ボタン → 3つの返信案カード → クリックで入力欄にセット
- **AI処理**: 直近の会話履歴 + ナレッジベース + 友だち情報をコンテキストに返信生成
- **実装量**: API 1エンドポイント ~80行、フロント ~150行
- **工数**: 中

---

### Phase 3: テーブルステークスの穴埋め（1-2セッション）
> 競合比較表で「×」を減らす

#### 3-1. カルーセル・ボタンメッセージ（A5）
- **仕様**: テンプレートにcarousel/buttonsタイプを追加
- **LINE API**: template messageのbuttons/carousel/confirm + Flex Message
- **UI**: テンプレート作成画面にタイプ選択 → カルーセルビルダー（カード追加/画像/タイトル/ボタン設定）
- **実装量**: API テンプレート拡張 ~100行、フロント カルーセルビルダー ~350行
- **工数**: 中

#### 3-2. URLクリック測定
- **仕様**: 配信メッセージ内のURLを自動的にトラッキングURL化（リダイレクト方式）
- **DB**: tracked_urls テーブルは既存、url_clicks も既存
- **処理**: メッセージ送信時にURL検出→短縮URL生成→クリック時にリダイレクト+記録
- **UI**: 分析ページにクリック率グラフ追加
- **実装量**: API リダイレクトエンドポイント + 送信時のURL置換 ~120行、フロント ~100行
- **工数**: 中（既存テーブル活用で軽い）

#### 3-3. コンバージョン測定
- **仕様**: 特定のURL到達やフォーム回答をCVとして計測
- **設定**: CV目標の定義（URL到達 / フォーム回答 / タグ付与）
- **集計**: CV率 = CV数 / 配信数 を自動計算
- **UI**: 分析ページにCV率ダッシュボード
- **実装量**: DB CVゴール定義テーブル ~30行、API ~100行、フロント ~200行
- **工数**: 中

#### 3-4. 対応ステータス拡張
- **現状**: unread / confirmed の2値
- **拡張**: unread / in_progress / done / needs_followup のenum化
- **UI**: チャットサイドバーのフィルタ拡張 + ステータス変更ドロップダウン
- **実装量**: DB ALTER ~5行、API ~20行、フロント ~50行
- **工数**: 小

---

### Phase 4: デモの武器を増やす（2セッション）
> 代理店が「これ見せたらクライアント食いつく」と思う機能

#### 4-1. AIオンボーディングウィザード
- **仕様**: 初回ログイン時に「業種は？」「目的は？」→ AIが全設定を一括生成
- **生成対象**: あいさつメッセージ / リッチメニュー構成 / ステップ配信5通 / 回答フォーム / タグセット
- **差別化**: Linyの「設計→構築→運用」を人間がやる部分を、AIが初回セットアップで全自動
- **実装量**: API ~300行（各モジュール連携）、フロント ウィザードUI ~500行
- **工数**: 大

#### 4-2. AIレポート自動生成（B5）
- **仕様**: 週次/月次で分析データを自然言語レポート化
- **出力**: 「今週の友だち追加数は前週比+12%。配信Aの開封率が低下傾向のため、Bパターンへの切替を推奨」
- **UI**: ダッシュボードに「AI週次レポート」カード
- **実装量**: API ~150行、フロント ~200行
- **工数**: 中

#### 4-3. ガチャ機能
- **仕様**: LINE上で「1日1回ガチャ」→ クーポン or ポイント付与
- **Liny事例**: 友だちの38%がタップ → 来場 → 受注。不動産で5,000万円の売上
- **実装**: Flex Messageでガチャ演出 → Webhook回答でクーポン自動付与
- **実装量**: API ~150行、フロント 設定UI ~200行
- **工数**: 中

#### 4-4. 離脱防止ポップアップ（JSスニペット）
- **仕様**: Webサイトに1行のscriptタグを埋め込む → 離脱検知時にLINE友だち追加ポップアップ
- **配布**: 設定画面で「このコードをサイトに貼ってください」とスニペット表示
- **実装量**: JSウィジェット ~100行、API設定 ~30行
- **工数**: 小

---

### Phase 5: 分析を深める（1-2セッション）
> 代理店がROI説明に使える分析機能

#### 5-1. クロス分析
- **仕様**: タグ × ステータス × 流入経路 のクロス集計テーブル
- **UI**: 行・列に任意のディメンションを設定 → ヒートマップ表示
- **実装量**: API ~150行、フロント ~300行
- **工数**: 中

#### 5-2. ファネル分析
- **仕様**: 友だち追加 → アンケート回答 → 来店予約 → 成約 の各ステップの遷移率
- **UI**: ファネルチャート（各ステップの人数と遷移率%）
- **実装量**: API ~100行、フロント ~250行
- **工数**: 中

#### 5-3. 流入経路分析（詳細版）
- **仕様**: QRコード/URL別に友だち追加数・CV率・売上を集計
- **現状**: traffic_sources テーブルは存在、基本的な流入追跡あり
- **拡張**: 流入元 × 期間 × CV目標のクロス集計
- **実装量**: API ~100行、フロント ~200行
- **工数**: 中

---

### Phase 6: 代理店向け機能（2-3セッション）
> 複数クライアントを管理する代理店の業務効率化

#### 6-1. 代理店ダッシュボード（C1）
- **仕様**: 1つのログインで全クライアントのアカウントを俯瞰
- **表示**: クライアント別の友だち数/配信数/CV数/未対応チャット数
- **実装量**: DB テナント階層化、API ~200行、フロント ~400行
- **工数**: 大

#### 6-2. 業種別初期設定テンプレート（C4）
- **仕様**: 「美容サロン」「不動産」「飲食」等を選ぶと、あいさつ/リッチメニュー/ステップ配信/フォーム が事前設定された状態でアカウント作成
- **Phase 4-1のオンボーディングウィザードと統合可能**
- **実装量**: テンプレートデータ定義 ~200行、API ~100行
- **工数**: 中
